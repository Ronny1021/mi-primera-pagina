#  Mi primera pagina

Este apartado contiene el desarrollo teórico sobre las tecnologías fundamentales de la web (HTML, CSS, JavaScript) y el uso de herramientas de desarrollo.

---

## Preguntas y Respuestas

<details>
<summary><b>1. ¿Qué es HTML y cuál es su función en la web?</b></summary>
Es el lenguaje estándar para estructurar el contenido de las páginas web mediante etiquetas que indican al navegador cómo mostrar textos, imágenes y vídeos. Es la base estructural que trabaja junto a CSS y JavaScript.
</details>

<details>
<summary><b>2. ¿Qué es una etiqueta HTML y menciona las etiquetas más comunes?</b></summary>
Es un código especial entre corchetes (`< >`) que define cómo mostrar el contenido. 
* **Etiquetas comunes:** `<html>`, `<head>`, `<title>`, `<body>`, `<h1>-<h6>`, `<p>`, `<a>`, `<img>`, `<div>`, `<ul>`, `<li>`.
</details>

<details>
<summary><b>3. ¿Qué es un atributo de una etiqueta HTML y menciona los más comunes?</b></summary>
Es información adicional dentro de una etiqueta que modifica su comportamiento o apariencia.
* **Atributos comunes:** `class`, `id`, `href`, `src`, `alt`, `style`, `target`.
</details>

<details>
<summary><b>4. ¿Qué es CSS y cómo se utiliza para el diseño web?</b></summary>
Es el lenguaje de estilos utilizado para definir colores, fuentes, tamaños y posiciones de los elementos HTML, permitiendo que las páginas sean atractivas y responsivas.
</details>

<details>
<summary><b>5. ¿Qué es una propiedad en CSS y menciona las propiedades más comunes?</b></summary>
Es un parámetro específico que define un aspecto del estilo (color, tamaño, etc.).
* **Propiedades comunes:** `color`, `background-color`, `font-size`, `width`, `padding`, `margin`, `display`.
</details>

<details>
<summary><b>6. ¿Qué es un selector en CSS y cuáles tipos existen?</b></summary>
Es el patrón usado para elegir a qué elementos HTML aplicarles estilo.
* **Tipos:** Universal (`*`), de Etiqueta (`p`), de Clase (`.clase`), de ID (`#id`), de Atributo (`[]`) y combinadores.
</details>

<details>
<summary><b>7. ¿Qué es JavaScript y cómo añade interactividad?</b></summary>
Es un lenguaje de programación que permite modificar el contenido y estilo de la web en tiempo real, manejando eventos como clics y animaciones sin recargar la página.
</details>

<details>
<summary><b>8. ¿Cuáles son los tipos de datos primitivos en JavaScript?</b></summary>
Son valores inmutables básicos del lenguaje:
* `string`, `number`, `boolean`, `null`, `undefined`, `bigint`, `symbol`.
</details>

<details>
<summary><b>9. ¿Cómo funcionan las estructuras de flujo (if, else, switch, bucles)?</b></summary>
Permiten tomar decisiones o repetir tareas. `if/else` gestiona condiciones, `switch` evalúa casos múltiples y los bucles (`for`, `while`) automatizan repeticiones.
</details>

<details>
<summary><b>10. ¿Por qué es importante usar nombres significativos para variables y métodos?</b></summary>
Mejora la legibilidad y mantenimiento del código, haciendo que sea fácil de entender para otros desarrolladores y para ti mismo en el futuro.
</details>

<details>
<summary><b>11. ¿Qué es una variable de entorno y por qué son importantes?</b></summary>
Son valores configurados fuera del código fuente para almacenar datos sensibles (claves API) o configuraciones que cambian entre desarrollo y producción.
</details>

<details>
<summary><b>12. ¿Qué son las herramientas de desarrollo de Chrome y cómo se accede?</b></summary>
Es un conjunto de utilidades integradas en el navegador para inspeccionar y depurar webs. Se accede con `F12` o `Ctrl+Shift+I`.
</details>

<details>
<summary><b>13. ¿Qué se puede hacer en el panel "Elements"?</b></summary>
Permite inspeccionar y modificar en tiempo real el HTML y CSS de la página sin alterar los archivos originales.
</details>

<details>
<summary><b>14. ¿Cómo se utiliza el panel "Console" y para qué es útil?</b></summary>
Se usa para ver errores, registrar mensajes (`console.log`) y ejecutar código JavaScript directamente para probar funcionalidades.
</details>

<details>
<summary><b>15. ¿Qué información se obtiene del panel "Network" y por qué es importante?</b></summary>
Muestra los recursos cargados (imágenes, scripts), sus tiempos de respuesta y errores de conexión. Es vital para optimizar la velocidad de la web.
</details>
